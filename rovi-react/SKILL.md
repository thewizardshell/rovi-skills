# ROVI — React

## Stack

- **Orval** for auto-generating API hooks from OpenAPI specs
- **TanStack Query** for server state (fetching, caching, mutations)
- **Axios** (preferred over fetch) as HTTP client
- **TanStack Router** for routing (file-based, type-safe)
- **Zustand or Jotai** for global client state
- **framer-motion** for animations (see rovi-design skill)

---

## Project Structure

```
src/
  api/
    mutator/
      custom-instance.ts   > Axios instance with interceptors
    model/                 > Orval generated types
    endpoints/             > Orval generated hooks (by tag)
  assets/
  components/              > Global reusable components
  core/
    layout/
    store/                 > Global store (Zustand/Jotai)
    providers/             > QueryClient, Router, Theme providers
  features/
    [FeatureName]/
      components/          > Feature-specific components
      hooks/               > Feature-specific hooks
      store/               > Feature-specific store (if needed)
      views/               > Feature pages/views
      types/               > Feature types
  hooks/                   > Global shared hooks
  utils/
  styles/
    global.css             > CSS variables (see rovi-design)
orval.config.ts            > Orval configuration
```

## Orval + Axios Setup

**Orval generates everything automatically.** You do NOT create `api/model/`, `api/endpoints/`, or any hook files manually. You only create two things by hand:

1. `orval.config.ts` — the configuration
2. `src/api/mutator/custom-instance.ts` — the Axios instance

Everything else (`model/`, `endpoints/`, types, hooks, query keys) is auto-generated by running `npx orval`. Never edit generated files.

**This requires the backend to expose a Swagger/OpenAPI spec.** See the backend skills (`rovi-fastify`, `rovi-nestjs`, `rovi-go`, `rovi-fastapi`) — Swagger documentation is mandatory for this workflow to work.

### Custom Axios Instance

File: `src/api/mutator/custom-instance.ts` (the only manual file in `api/`)

```typescript
import Axios from "axios";
import type { AxiosRequestConfig, AxiosError } from "axios";

const axios = Axios.create({
  baseURL: "http://localhost:8080",
});

axios.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem("token");
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error),
);

export const customInstance = <T>(
  config: AxiosRequestConfig,
  options?: AxiosRequestConfig,
): Promise<T> => {
  return axios({ ...config, ...options }).then(({ data }) => data);
};

export type ErrorType<Error> = AxiosError<Error>;
export type BodyType<BodyData> = BodyData;
```

### orval.config.ts

export default defineConfig({

### Generate and use

```bash
# Generate hooks from OpenAPI spec
npx orval

# Add to package.json scripts
# "generate:api": "orval"
```

Orval auto-generates typed hooks, query keys, fetcher functions, and TypeScript models. **Always use what Orval exposes** — it abstracts the entire API layer:

```typescript
// Everything is auto-generated: hooks, types, query keys, mutations
import { useListUsers, useCreateUser, useShowUserById } from "../api/endpoints/users/users";
import type { User, CreateUserBody } from "../api/model";

function UserList() {
  // useListUsers = auto-generated useQuery hook with typed response
  const { data, isLoading, error } = useListUsers();

  // useCreateUser = auto-generated useMutation with typed body + invalidation
  const { mutate: createUser, isPending } = useCreateUser();

  const handleCreate = (userData: CreateUserBody) => {
    createUser({ data: userData });
  };
}

function UserDetail({ id }: { id: string }) {
  // useShowUserById = auto-generated hook, already typed, already cached
  const { data: user } = useShowUserById(id);
}
```

**You never write `useQuery`/`useMutation` manually when Orval is configured.** The generated hooks already handle query keys, types, cache invalidation, and abort signals. Just import and use.

## Manual TanStack Query (when no OpenAPI spec)

If there is no OpenAPI spec and Orval cannot be used, write hooks manually:

```typescript
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import { customInstance } from "../../api/mutator/custom-instance";

export function useUsers() {
  return useQuery({
    queryKey: ["users"],
    queryFn: () => customInstance<User[]>({ url: "/users", method: "GET" }),
  });
}

export function useCreateUser() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: (data: CreateUserData) =>
      customInstance<User>({ url: "/users", method: "POST", data }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["users"] });
    },
  });
}
```

## TanStack Router (File-Based)

Routes live in a `routes/` folder. Each route is a folder with an `index.tsx` using `createFileRoute`.

```
src/
  routes/
    __root.tsx               > Root layout
    index.tsx                > /
    tasks/
      index.tsx              > /tasks
    login/
      index.tsx              > /login
    dashboard/
      index.tsx              > /dashboard
```

### Route Pattern

Every route file uses `createFileRoute` with:
- **`loader`** — prefetch data with `ensureQueryData` using Orval's generated query options
- **`pendingComponent`** — loading UI while data loads
- **`errorComponent`** — error UI with retry
- **`component`** — the actual page component

```typescript
import { createFileRoute } from "@tanstack/react-router";
import {
  useGetUsersUserIdTasks,
  getGetUsersUserIdTasksQueryOptions,
  usePostTasks,
  usePatchTasksIdStatus,
  useDeleteTasksId,
  getGetUsersUserIdTasksQueryKey,
  type DomainTask,
} from "@/api/endpoints";

export const Route = createFileRoute("/tasks")({
  loader: async ({ context }) => {
    const userId = Number(localStorage.getItem("userId") || "1");
    await context.queryClient.ensureQueryData(
      getGetUsersUserIdTasksQueryOptions(userId),
    );
  },
  pendingComponent: () => <LoadingSpinner text="Cargando tareas..." />,
  errorComponent: ({ error }) => <ErrorCard message={error.message} />,
  component: TasksComponent,
});

function TasksComponent() {
  const queryClient = useQueryClient();
  const userId = Number(localStorage.getItem("userId") || "1");

  // All hooks are Orval-generated — typed, cached, with query keys
  const { data: tasks } = useGetUsersUserIdTasks(userId, {
    query: { refetchOnWindowFocus: false, staleTime: 1000 * 60 * 5 },
  });

  const createMutation = usePostTasks({
    mutation: {
      onSuccess: () => {
        queryClient.invalidateQueries({
          queryKey: getGetUsersUserIdTasksQueryKey(userId),
        });
      },
    },
  });

  const updateStatusMutation = usePatchTasksIdStatus({
    mutation: {
      onSuccess: () => {
        queryClient.invalidateQueries({
          queryKey: getGetUsersUserIdTasksQueryKey(userId),
        });
      },
    },
  });

  const deleteMutation = useDeleteTasksId({
    mutation: {
      onSuccess: () => {
        queryClient.invalidateQueries({
          queryKey: getGetUsersUserIdTasksQueryKey(userId),
        });
      },
    },
  });

  // ... render with tasks, mutations
}
```

**Key points:**
- `loader` uses Orval's `get*QueryOptions()` to prefetch — data is ready before the component mounts
- Mutations use Orval's `use*` hooks with `onSuccess` to invalidate via Orval's `get*QueryKey()`
- Everything is typed end-to-end: request body, response, query keys — all from the OpenAPI spec

## Component Pattern

```typescript
// features/Users/components/UserCard.tsx
interface UserCardProps {
  user: User;
  onDelete: (id: string) => void;
}

export function UserCard({ user, onDelete }: UserCardProps) {
  return (
    <motion.div
      initial={{ opacity: 0, y: 20 }}
      whileInView={{ opacity: 1, y: 0 }}
      viewport={{ once: true }}
    >
      <h3>{user.name}</h3>
      <p>{user.email}</p>
      <button onClick={() => onDelete(user.id)}>Eliminar</button>
    </motion.div>
  );
}
```

## Rules

- **Orval first.** If there is an OpenAPI spec, always use Orval to generate hooks. No manual API code.
- **Axios over fetch.** Always use the custom Axios instance, never raw fetch.
- **TanStack Router file-based.** `routes/` folder, each route is a folder with `index.tsx`, `createFileRoute` with `loader`/`pendingComponent`/`errorComponent`.
- **Prefetch in loader.** Use `ensureQueryData` with Orval's `get*QueryOptions()` so data is ready before mount.
- **Invalidate with Orval's query keys.** Use `get*QueryKey()` in `onSuccess` — never hardcode query key strings.
- **TanStack Query for server state.** No fetching in useEffect. No manual cache management.
- **Global store for client state.** UI state, filters, selections — not server data.
- **One store per entity.** See rovi-store skill.
- **Function components only.** No class components.
- **Custom hooks for logic.** Components are for rendering, hooks are for logic.
- **No `fromPersistence` pattern.** Do not add static reconstruction methods unless explicitly requested.
